# Skills Index

This document lists the first official Caveman AI Skills included in this repository.

## Core skills

### project-discovery

Use before implementing, debugging, reviewing, refactoring, or documenting a project. Detects stack, architecture, package manager, database, test commands, build commands, and existing harness files.

### architecture-analyzer

Use to understand project architecture, module boundaries, data flow, dependencies, and design patterns without forcing a new structure.

### feature-implementation

Use when adding or modifying features. Focuses on minimal, safe, testable implementation that follows existing project conventions.

### bug-fixer

Use for defect investigation and fixes. Encourages reproduction, root-cause analysis, regression checks, and small safe changes.

## Engineering domain skills

### database-engineer

Use for schema, queries, migrations, indexes, transactions, ORM behavior, data integrity, and database performance.

### api-engineer

Use for REST, GraphQL, RPC, webhooks, validation, auth, response formats, pagination, and API compatibility.

### frontend-engineer

Use for frontend components, state, styling, accessibility, responsive behavior, UI forms, and client-side performance.

### testing-engineer

Use for unit, integration, API, database, UI, E2E, and regression tests.

### security-reviewer

Use for authentication, authorization, input validation, secrets, data exposure, file uploads, payments, webhooks, and sensitive operations.

### performance-reviewer

Use for backend, frontend, database, API, queue, memory, build, and runtime performance review.

### devops-deployment

Use for CI/CD, Docker, servers, environments, queues, workers, cloud services, logs, releases, and rollback planning.

## Workflow skills

### git-pr-workflow

Use for branches, commits, pull requests, conflict handling, PR summaries, rollback notes, and safe Git guidance.

### documentation-writer

Use for README files, setup guides, API docs, architecture docs, release notes, onboarding guides, and troubleshooting docs.

### code-reviewer

Use for reviewing changes for correctness, maintainability, security, performance, tests, architecture fit, and PR readiness.

## Safety principle

Every skill must respect the same rule:

> Project-specific harnessing always overrides universal skills.
