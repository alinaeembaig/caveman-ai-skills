# Security Policy

Caveman AI Skills is a public skill and documentation repository. It should not contain secrets, credentials, private project data, or sensitive customer information.

## Reporting a vulnerability

If you find a security concern in this repository, please open a GitHub issue with a clear description unless the report contains sensitive details.

For sensitive reports, contact the repository owner privately through GitHub.

## Security expectations for contributions

Contributions must not include:

- API keys
- access tokens
- passwords
- private keys
- private customer data
- internal company data
- proprietary project code copied without permission

## Skill safety rule

Security-sensitive skills must always instruct agents to:

- follow project-specific security rules first
- avoid hardcoded credentials
- avoid exposing sensitive data in logs or responses
- preserve authentication and authorization checks
- explain security risks clearly

## AI usage warning

AI-generated code or instructions should be reviewed by a human before production use.
